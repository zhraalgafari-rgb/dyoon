# خطة تطوير: الاستيراد، المرفقات، وكشف الحساب الاحترافي

## المرحلة 1 — تحسين الاستيراد (Excel / PDF)

**الهدف:** استيراد أكثر ذكاءً ومرونة مع معاينة دقيقة.

- **Excel متعدد الأوراق:** اختيار الورقة (Sheet) قبل التحليل بدل الاكتفاء بالأولى.
- **كشف الترويسة تلقائياً:** تخطي صفوف العنوان/الشعار قبل صف الأعمدة.
- **AI Mapping محسّن:** إرسال 10 صفوف عينة بدل 3، وحفظ آخر تخطيط استخدمه المستخدم لإعادة استعماله.
- **PDF محسّن:** 
  - استخراج جداول حقيقية عبر `page.getTextContent` مع تحليل إحداثيات x/y لإعادة بناء الأعمدة.
  - دعم PDF ممسوح (OCR اختياري عبر Tesseract.js عند الفشل).
- **معاينة تفاعلية:** جدول قابل للتعديل خلية بخلية قبل الحفظ + تمييز الأخطاء بلون أحمر.
- **كشف التكرارات:** مطابقة فازية بالاسم/الجوال لاقتراح "دمج مع عميل موجود".
- **استيراد العملات:** مطابقة رمز العملة تلقائياً مع الموجودة + إنشاء جديدة عند الحاجة.
- **التراجع (Undo):** حفظ batch_id لكل عملية استيراد + زر "تراجع عن آخر استيراد".

## المرحلة 2 — التصدير الاحترافي

- **Excel متعدد الأوراق منسّق:** ألوان رؤوس، تجميد الصف الأول، إجماليات تلقائية، تنسيق أرقام `#,##0.00`.
- **قوالب جاهزة:** "كشف حساب"، "تقرير شهري"، "ملخص ديون"، "تقرير مصاريف".
- **CSV/JSON** للنسخ الاحتياطي الكامل.

## المرحلة 3 — مرفقات العملاء (فواتير وسندات)

**الهدف:** ربط ملفات/صور بكل عميل ومعاملة.

- **توسيع جدول `attachments`:** إضافة `person_id` (إلى جانب `transaction_id` الموجود) للسماح بمرفقات على مستوى العميل.
- **تبويب "المرفقات" في صفحة العميل:** شبكة صور/أيقونات PDF مع:
  - رفع متعدد (Drag & Drop + التقاط من الكاميرا على الموبايل).
  - تصنيف: فاتورة / سند قبض / سند صرف / عقد / هوية / أخرى.
  - ملاحظة + تاريخ + مبلغ اختياري.
  - معاينة (Image lightbox / PDF viewer).
  - تنزيل، مشاركة عبر واتساب، حذف.
- **ضغط الصور** قبل الرفع (browser-image-compression) لتقليل الاستهلاك.
- **حماية:** RLS صارم + روابط موقّعة (Signed URLs) قصيرة العمر.
- **إحصائيات:** عدد المرفقات وحجمها في رأس صفحة العميل.

## المرحلة 4 — كشف الحساب الاحترافي (Statement v2)

**الهدف:** كشف حساب بمستوى الأنظمة المحاسبية الكبرى، ثنائي العملة (ر.س / ر.ي).

### التصميم البصري
- **رأس فاخر:** شريط متدرج بهوية التطبيق + شعار المنشأة + بيانات الاتصال + رقم ضريبي.
- **بطاقة العميل:** اسم، جوال، رقم العميل، تاريخ الفتح، تاريخ الكشف.
- **ملخص علوي (KPI):** 4 بطاقات — إجمالي له، إجمالي عليه، الصافي، عدد المعاملات — لكل عملة.

### الجدول التفصيلي
- أعمدة: التاريخ | الوصف | مرجع | مدين | دائن | الرصيد الجاري | العملة.
- **رصيد افتتاحي** ظاهر بوضوح في أول صف.
- **ترقيم تلقائي** للحركات.
- **صفوف ملوّنة بالتناوب** + تمييز المدفوع/المتأخر.
- **مجاميع جزئية شهرية** (اختياري).
- **إجمالي ختامي** ملوّن بحسب الحالة (أخضر=له، أحمر=عليه).

### دعم ثنائي العملة
- **قسم منفصل لكل عملة** (ر.س ثم ر.ي) مع جدول مستقل ورصيد ختامي خاص.
- **قسم موحّد بالعملة الأساسية** يحول كل شيء حسب سعر الصرف التاريخي (موجود مسبقاً).
- خيار في الواجهة: "عرض بكل العملات" / "عرض بالعملة الأساسية فقط".

### تنسيقات التصدير
- **PDF:** A4 احترافي بـ jsPDF + autoTable (الأساس موجود — يُعاد تصميمه).
- **Excel:** ورقة لكل عملة + ورقة ملخص + تنسيق ألوان وحدود.
- **صورة (PNG):** لقطة من الكشف للمشاركة السريعة عبر واتساب.
- **رابط مشاركة:** رابط موقّع للكشف يفتح في المتصفح (اختياري).

### واجهة بناء الكشف
- **اختيار الفترة:** الكل / هذا الشهر / آخر 3 شهور / مخصص (من-إلى).
- **خيارات:** إظهار المدفوع فقط، إخفاء المعاملات الصفرية، تضمين المرفقات (صور مصغرة).
- **معاينة مباشرة قبل التصدير.**
